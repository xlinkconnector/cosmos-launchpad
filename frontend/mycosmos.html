Installation Methods

Direct Installation

Download and install the complete package with full control over configuration.

Download Package

Download the latest release package to your server.

wget https://releases.cosmos-ultimate.com/cosmos-ultimate-2.0.0-latest.tar.gz

Verify Package

Verify the package integrity using SHA256 checksum.

sha256sum cosmos-ultimate-2.0.0-latest.tar.gz

Extract and Install

Extract the package and run the installation script.

tar -xzf cosmos-ultimate-2.0.0-latest.tar.gz
cd cosmos-ultimate-2.0.0
sudo ./install_cosmos_ultimate.sh

Configure Deployment

Set deployment level and optional features during installation.

export DEPLOYMENT_LEVEL="full" # minimal, standard, or full
export ENABLE_AI_FEATURES=true
export ENABLE_ANALYTICS=true
export COSMOS_DOMAIN="your-domain.com"

One-Line Installation

Fastest way to get started - downloads and installs everything automatically.

Quick Install

Single command installs the complete platform with default settings.

curl -sSL https://install.cosmos-ultimate.com | sudo bash

Custom Configuration

Install with custom configuration using environment variables.

export COSMOS_DOMAIN="your-domain.com" && curl -sSL https://install.cosmos-ultimate.com | sudo bash

Docker Deployment

Containerized deployment for easy management and scaling.

Pull Image

Download the official Docker image from the registry.

docker pull cosmos-ultimate/platform:latest

Run Container

Start the platform with port mapping and volume mounts.

docker run -d \
  --name cosmos-ultimate \
  -p 80:80 -p 443:443 \
  -v cosmos-data:/opt/cosmos-ultimate/data \
  -e COSMOS_DOMAIN="your-domain.com" \
  cosmos-ultimate/platform:latest

Docker Compose

Use Docker Compose for multi-service deployment with databases.

curl -O https://releases.cosmos-ultimate.com/docker-compose.yml
docker-compose up -d

Kubernetes Deployment

Production-scale deployment with auto-scaling and high availability.

Helm Repository

Add the official Helm repository for Kubernetes deployment.

helm repo add cosmos-ultimate https://charts.cosmos-ultimate.com
helm repo update

Install Chart

Deploy using Helm with custom values for your environment.

helm install cosmos-ultimate cosmos-ultimate/platform \
  --set ingress.domain=your-domain.com \
  --set replicas=3 \
  --set autoscaling.enabled=true

Manual Deployment

Deploy using raw Kubernetes manifests for custom configurations.

kubectl apply -f https://releases.cosmos-ultimate.com/k8s/namespace.yaml
kubectl apply -f https://releases.cosmos-ultimate.com/k8s/deployment.yaml
kubectl apply -f https://releases.cosmos-ultimate.com/k8s/service.yaml

Multi-Cloud Deployment

Enterprise deployment across AWS, GCP, and Azure with Terraform automation.

Terraform Setup

Download and configure Terraform modules for cloud deployment.

git clone https://github.com/cosmos-ultimate/terraform-modules.git
cd terraform-modules
terraform init

Configure Variables

Set up terraform.tfvars with your cloud provider credentials and preferences.

cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your settings
terraform plan

Deploy Infrastructure

Deploy the complete infrastructure across multiple cloud providers.

terraform apply
# Deployment typically takes 15-20 minutes
terraform output cosmos_dashboard_url

System Requirements

🔧 Hardware Requirements

  • Minimal: 4GB RAM, 2 CPU cores, 50GB storage
  • Standard: 8GB RAM, 4 CPU cores, 100GB storage
  • Full: 16GB RAM, 8 CPU cores, 500GB storage
  • Enterprise: 32GB+ RAM, 16+ CPU cores, 1TB+ storage

💻 Operating System

  • AlmaLinux 8.x, 9.x (Recommended)
  • RHEL 8.x, 9.x
  • CentOS Stream 8, 9
  • Ubuntu 20.04, 22.04 LTS
  • Debian 11, 12
  • Fedora 36, 37, 38

🌐 Network Requirements

  • Internet connection for package downloads
  • Open ports: 80 (HTTP), 443 (HTTPS)
  • Optional: 26656-26660 (Cosmos P2P)
  • Firewall configuration support
  • DNS resolution for domain setup

🔑 Access Requirements

  • Root or sudo privileges
  • SSH access to server
  • Package manager access (dnf/yum/apt)
  • Internet access during installation
  • SSL certificate domain control

☁️ Cloud Compatibility

  • AWS EC2 (all instance types)
  • Google Cloud Compute Engine
  • Microsoft Azure Virtual Machines
  • DigitalOcean Droplets
  • Bare metal servers
  • VPS providers

🐳 Container Support

  • Docker 20.x or later
  • Docker Compose 2.x
  • Kubernetes 1.24+ (optional)
  • Helm 3.x (for K8s deployment)
  • Container runtime (containerd/CRI-O)
